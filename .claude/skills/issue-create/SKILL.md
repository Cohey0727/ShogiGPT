# Issue Create

GitHub Issueを作成する。

## 手順

### 1. 情報収集
ユーザーから以下を聞く（わかる範囲で）:
- 症状（何が起きているか）
- 再現手順
- 期待する動作
- 既に試したこと

### 2. 自動調査
可能であれば:
- 関連するログファイル (`data/logs/`) を確認
- 関連するコードを特定
- エラーメッセージを取得

### 3. Issue作成
```bash
gh issue create --repo Cohey0727/ShogiGPTApp \
  --title "<簡潔なタイトル>" \
  --body "$(cat <<'EOF'
## 症状
<何が起きているか>

## 再現手順
```bash
<再現コマンド>
```

## 期待する動作
<本来どうなるべきか>

## 実際の動作
<実際に何が起きたか>

## 調査済み
- <調べたこと>
- <試したこと>

## 関連ファイル
- <関連するソースコード>

## 環境
- モデル: <使用モデル>
- OS: macOS (Apple Silicon)
EOF
)"
```

### 4. ラベル付け（任意）
```bash
gh issue edit <番号> --add-label "bug"
```

## ルール
- タイトルは簡潔に（何が壊れているか一目でわかるように）
- 再現手順は具体的に（他の人/LLMが再現できるように）
- 調査済みの内容は必ず書く（同じ調査を繰り返さないように）
